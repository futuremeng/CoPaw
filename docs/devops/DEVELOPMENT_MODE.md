# CoPaw 开发模式（数字出版方向）

## 1. 仓库定位
- 聚焦数字出版领域，优先支持行业相关需求、创新与定制化。

## 2. 反馈与任务分类标准
- 行业相关需求/创新/定制化：本地优先处理，积累特色能力。
- 框架基本特性和 bug：本地优先修复，影响开发的先解决再提 PR 到源库。
- 一般性问题（无行业特色、无本地影响）：直接提交到源库 issues。

## 3. 工作流程
1. 收集反馈（issue、社区建议、用户需求等）。
2. 分类归属（本地/源库），标明处理策略：
   - 本地干：行业相关、关键 bug。
   - 本地修好再 PR：框架 bug、影响开发的特性。
   - 直接提源库 issue：一般性问题。
3. 路线图、任务列表、issue、PR 等条目需标明归属与处理策略。
4. 本地优先处理行业相关和关键 bug，修复后同步 PR。
5. 一般性问题直接同步到源库 issues，保持与上游协作。
6. 定期梳理和同步，确保本地与源库协同推进。

## 4. 模板建议
- issue/task/PR 模板增加字段：
  - 归属：本地/源库
  - 处理策略：本地干/本地修好再 PR/直接提源库 issue
  - 领域相关性：数字出版/通用/其他
