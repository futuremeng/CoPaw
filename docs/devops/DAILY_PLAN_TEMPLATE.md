## 每日开发计划流程

1. 同步 upstream 路线图与最新 PR/issues
2. 梳理 upstream 近期 PR、issue、milestone，记录关键进展与标准
3. 检查本地 ROADMAP.md 与 FORK_ROADMAP.md，补充/调整 fork 特性
4. 制定当天开发任务，优先保证兼容与同步，适当等待 upstream 标准落地
5. 有成果时，主动贡献 PR 给 upstream，记录贡献计划
6. 记录当天开发完成情况与遗留问题

## 示例 Checklist

- [ ] 同步 upstream 路线图
- [ ] 梳理 upstream PR/issues

- [ ] 制定当天开发任务
- [ ] 贡献计划与兼容性检查
- [ ] 记录开发完成情况
